# 🛠️ Track A: Applied Engineering (Michigan Curriculum)

> **Academic Focus:** Implementation of scalable Data Science pipelines for Anti-Money Laundering (AML) and ESG Compliance.  
> **Curriculum Basis:** Applied Data Science with Python (University of Michigan).

Dieser Track dokumentiert die programmatische und handwerkliche Umsetzung regulatorischer Analytics-Pipelines. Der Fokus liegt auf der Anwendung nativer Python-Bibliotheken (NumPy, Pandas, Scikit-Learn, NetworkX) zur Verarbeitung, Analyse und Modellierung hochdimensionaler Finanzdaten.

## 📂 Modul-Architektur & Domänen-Transfer

Das Curriculum ist in fünf konsekutive Anwendungsbereiche strukturiert, die direkt auf die Anforderungen der forensischen Datenanalyse zugeschnitten sind:

1. **`01_Intro_to_Data_Science`**: Fundamentale Datenmanipulation (Regex, NumPy, Pandas) und Implementierung von Privacy Engineering (Differential Privacy).
2. **`02_Applied_Plotting`**: Visuelle Repräsentation statistischer Anomalien und Generierung von Compliance-Dashboards.
3. **`03_Applied_Machine_Learning`**: Anwendung prädiktiver Modelle (Supervised Learning) zur Automatisierung von Risiko-Klassifizierungen (KYC/AML).
4. **`04_Applied_Text_Mining`**: Syntaktische und semantische Extraktion von Metadaten aus unstrukturierten Audit-Logs und SWIFT-Transaktionen.
5. **`05_Applied_Social_Network_Analysis`**: Graphentheoretische Modellierung von Finanznetzwerken zur Detektion organisierter Geldwäsche-Topologien (z. B. Smurfing, zyklische Transaktionen).

---
*Dieser Track bildet das methodische Fundament (Engineering) für die tiefergehenden mathematischen Analysen in Track B (Algorithmic Theory).*
